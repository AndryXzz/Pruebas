En HTML5 no es necesario proporcionar valores para todos los atributos, como los controles, reporoducción automática y atributos de bucle utilizados con el elemente <video>. Estos atributos son como interruptores de encendido/apagado. Si el atributo está presente, actica esa opción. si se omite el atributo, la opción está desactivada. si el navegador no admite el elemento <video> o el formato de video utilizado . se mostrará lo que esté entre la <video> de apretura y las etiquetas de </video> de cierre.

preload: este articulo indica al navegador qué hacer cuando se carga la página. Puede tener uno de tres valores:
none:el navegador no debe cargar el vídeo hasta que el usuario presiona play.
auto:el navegador debe descargar el vídeo cuando se carga la página.
metadata: el navegador debe simplemente recopilar información como el tamaño, el primer fotograma, la lista de pistas y duración.

La etiqueta video: para especificar la ubicación del archivo que se reproducirá puede usar el elemento <source> dentro del elemento <video>. (Esto debe reemplazar el atributo src en la etiqueta <video> de apertura.) También puede utilizar varios elementos <source> para especificar que el vídeo está disponible en diferentes formatos. (Debido a un error en el iPad, debe proporcionar el video MP4 como el primer formato. De lo contrario, puede que no inicie,)

src: este atributo especifica la ruta de acceso al vídeo.

type: se debe utilizar este atributo para decirle al navegador qué formato es el vídeo. De lo contrario, descargará parte del video para ver si puede reproducir el archivo (que tomará tiempo y ancho de banda).

codecs: el cídec qe se usó para codificar el vídeo se suministra dentro del atributo type. TEnga en cuenta el uso de comillas simples, así como comillas dobles en el atributo de tipo, cuando se suministra

Actividad 3


<audio> HTML5 introdujo el elemento <audio> para incluir archivos de audio en sus páginas. Al igual que con el vídeo HTML5, los navegadores esperan diferentes formatos para el audio. El elemento <audio> tiene un número de atributos que le permiten controlar la reproducción de audio.

src: este atributo especifica la rita de acceso al archivo de audio.

controls: este atributo indica si el reproductor debe mostrar los controles. si no utiliza este atributo, no se mostrarán controles de forma predeterminada.

loop: este atributo especifica que la pista de audio debe reproducirse de nuevo una vez haya finalizado

Youtube video ID Youtube mostrará un UD (como EF81Pb5IWoU), al guardar (o reproducir) un vídeo. puede utilizar este ID y hacer referecia a su vídeo en el código HTML.

reproducción de un vídeo de Youtube en HTML:
Para reproducir el vídeo en una página web, haga lo siguiente:

  • Sube el vídeo a youtube
  • Anote el ID del vídeo
  • Defina un elemento <iframe> en su página web
  • Deje que el atributo src apunto a la URL del Vídeo
  • Utilice los atributos width y height para especificar la dimensión del reproductor.
